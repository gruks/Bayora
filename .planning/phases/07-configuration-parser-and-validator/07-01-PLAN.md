  <name>Create Policy models, Registry, and parser logic</name>
  <files>packages/centinela-core/src/centinela/config/policy.py</files>
  <action>
Create policy.py defining:
- `ResourceLimitsOverride`: BaseModel allowing optional resources (CPU, memory, timeout, disk)
- `SecurityConfigOverride`: BaseModel allowing optional security settings
- `SecurityPolicy`: BaseModel representing a policy, containing name, description, extends, resources, security, and dependencies
- `CircularDependencyError`: Custom exception
- `detect_cycles`: Function to detect circular references in any directed graph represented as an adjacency list
- `PolicyRegistry`: Class storing raw parsed policies, running cycle detection, and resolving inheritance:
  - `register(policy)`: Adds a policy
  - `resolve(name)`: Returns a resolved `PlatformConfig` by merging overrides with parent policies recursively
  - Enforces merged CPU cores (0.1–32) and memory (128–65536 MB) on the resolved `PlatformConfig`
- `PolicyParser`: Utility to load policies from YAML/JSON strings or files, applying env var substitution before Pydantic parsing
  </action>
  <verify>Code imports correctly and defines the classes with correct signatures</verify>
  <done>policy.py implemented and verified</done>
</task>
