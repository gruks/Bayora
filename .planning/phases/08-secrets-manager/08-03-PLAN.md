# Phase 8, Plan 3: ABAC Policy DSL for Secrets

## Objective
Implement Attribute-Based Access Control (ABAC) to enforce which containers/tenants can access specific secrets.

## Requirements
1. Define a DSL (Domain Specific Language) in YAML for defining access policies (e.g., `allow read on secret:api_key if tenant_role == red-agent`).
2. Integrate with the `PolicyParser` from Phase 7.
3. Implement `ABACEngine` in `centinela.secrets.abac` that takes a policy, a user/tenant context, and evaluates whether access is granted.
4. Wrap `SecretStore` with a proxy `SecureSecretStore` that enforces ABAC before forwarding calls to the underlying store.

## Steps
1. **Schema Definition**: Define Pydantic models for ABAC policies (Rule, Subject, Resource, Action) in `packages/centinela-core/src/centinela/secrets/abac_models.py`.
2. **Engine Implementation**: Create `ABACEngine.evaluate(subject_attrs, resource_attrs, action)` returning a boolean.
3. **Secure Proxy**: Implement `SecureSecretStore(SecretStore)` that takes an `ABACEngine` and an underlying `SecretStore`.
4. **Testing**: Write test cases covering various policy scenarios (allow/deny based on attributes).

## Definition of Done
- Pydantic models for ABAC policies validate correctly.
- `ABACEngine` correctly enforces rules.
- `SecureSecretStore` properly gates access to `get_secret` and `set_secret`.
