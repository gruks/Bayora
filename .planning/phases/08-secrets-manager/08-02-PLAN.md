# Phase 8, Plan 2: tmpfs Secret Injection Service

## Objective
Provide a secure mechanism to mount and inject secrets into containers using tmpfs memory partitions so secrets never touch disk.

## Requirements
1. Implement a `SecretInjector` class.
2. Given a `SecretRef`, retrieve the secret from `SecretStore`.
3. Mount a tmpfs volume (or verify it is mounted) at a specified path (e.g. `/var/run/secrets/`).
4. Write the decrypted secret to a file in the tmpfs path with `0400` permissions.
5. Provide a cleanup mechanism to wipe the file and unmount the tmpfs volume when the session terminates.

## Steps
1. **Create Injector**: In `packages/centinela-core/src/centinela/secrets/injector.py`, create `SecretInjector`.
2. **tmpfs Operations**: Implement methods to write to the designated path, utilizing `os.chmod` to enforce `0400`.
3. **Cleanup Protocol**: Implement a `cleanup(secret_ref: SecretRef)` method which uses `os.remove()` and ensures memory is zeroed out if possible, though unlinking from tmpfs generally frees it.
4. **Testing**: Write unit tests in `tests/secrets/test_injector.py`. (Use a local temporary directory to simulate tmpfs for tests).

## Definition of Done
- `SecretInjector` can successfully fetch, write, and clean up secrets.
- Unit tests pass.
