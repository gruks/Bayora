# Phase 8, Plan 1: Storage Backends (SQLite & Vault)

## Objective
Implement concrete storage backends for the `SecretStore` ABC defined in Phase 5.

## Requirements
1. Implement `SQLiteSecretStore` which stores secrets in a SQLite database. All payloads must be encrypted using `SecretManager` before insertion.
2. Implement `VaultSecretStore` which interacts with HashiCorp Vault using the `hvac` library.
3. Update `pyproject.toml` to include `hvac` and `aiosqlite` or `sqlalchemy` if needed (already included in phase 5).
4. Provide unit tests with mock backends.

## Steps
1. **Add Dependencies**: Ensure `hvac` is added to `pyproject.toml` in `packages/centinela-core` and run `uv sync`.
2. **Implement SQLiteStore**: In `packages/centinela-core/src/centinela/secrets/sqlite_store.py`, create `SQLiteSecretStore(SecretStore)` that sets up a local DB and handles CRUD of secrets using `SecretManager.encrypt/decrypt`.
3. **Implement VaultStore**: In `packages/centinela-core/src/centinela/secrets/vault_store.py`, create `VaultSecretStore(SecretStore)` utilizing `hvac.Client` for KV v2 secrets engine.
4. **Testing**: Add `tests/secrets/test_sqlite_store.py` and `test_vault_store.py` (mocking Vault).

## Definition of Done
- `SQLiteSecretStore` and `VaultSecretStore` are fully implemented and pass unit tests.
- Payloads in SQLite are verified to be stored as encrypted ciphertext.
