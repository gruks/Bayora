# Phase 8, Plan 4: Automatic Key Rotation & mTLS Setup

## Objective
Provide automated 24-hour key rotation for the master encryption key and establish mutual TLS (mTLS) for orchestrator/agent communication.

## Requirements
1. **Rotation**: Implement a background service/task that generates a new master key, iterates over all secrets, decrypts them with the old key, encrypts with the new key, and updates the `SecretStore`.
2. **mTLS Setup**: Provide utility scripts/functions to generate an internal Certificate Authority (CA) and issue client/server certificates.
3. **FastAPI integration**: Add configurations to FastAPI to enforce mTLS validation using the generated CA.

## Steps
1. **Key Rotation Task**: In `packages/centinela-core/src/centinela/secrets/rotation.py`, create `KeyRotator(store: SecretStore, manager: SecretManager)`.
2. **mTLS Utilities**: In `packages/centinela-core/src/centinela/secrets/mtls.py`, use the `cryptography` library to implement `generate_ca()`, `generate_server_cert()`, and `generate_client_cert()`.
3. **Testing**: Write tests validating that secrets are readable after rotation and mTLS certs are correctly signed by the generated CA.

## Definition of Done
- Key rotation correctly updates all secrets without data loss.
- X.509 CA and certificate generation functions work and output valid PEM files.
- Tests pass.
